## 🎯 اختبار #1: إنشاء مستخدم Admin

### الهدف:
إنشاء مستخدم Admin الأول في النظام

### الخطوات:
```bash
1. افتح Terminal/PowerShell
2. اذهب للمجلد: cd c:\xampp\htdocs\studentManagement
3. اكتب: php artisan tinker
4. انسخ والصق:
```

```php
User::create([
    'name' => 'Admin',
    'email' => 'admin@test.com',
    'password' => Hash::make('password123'),
    'role' => 'admin'
])
```

### النتيجة المتوقعة:
```
✅ إنشاء مستخدم جديد مع role = 'admin'
✅ يظهر رسالة: User {#...}
✅ البيانات محفوظة في قاعدة البيانات
```

### التحقق:
```bash
اكتب في tinker:
User::where('email', 'admin@test.com')->first()
يجب أن تشاهد:
- name: "Admin"
- email: "admin@test.com"
- role: "admin"
```

## 👤 اختبار #3: التسجيل كـ User جديد

### الهدف:
التحقق من أن التسجيل الجديد يعطي دور 'user' افتراضياً

### الخطوات:
```
1. اذهب إلى: http://localhost/studentManagement
2. اضغط "Register"
3. أدخل البيانات:
   - Name: أحمد محمد
   - Email: user1@test.com
   - Password: password123
   - Confirm Password: password123
4. اضغط Register
```

### النتيجة المتوقعة:
```
✅ إنشاء مستخدم جديد
✅ الدور = 'user' تلقائياً
✅ إعادة توجيه إلى صفحة الدخول
```

### الخطوة اللاحقة:
```
5. أدخل: user1@test.com / password123
6. اضغط Login
```

### النتيجة المتوقعة بعد الدخول:
```
✅ إعادة توجيه إلى: /user/courses
✅ ظهور "Available Courses" كـ Page Title
✅ عرض الدورات المتاحة
✅ الـ Navbar يحتوي على: Available Courses, My Enrollments, Profile
✅ NO Dashboard, NO Students, NO Teachers links
```

## 🔑 اختبار #8: Logout الخروج

### الهدف:
التحقق من أن زر الخروج يعمل بشكل صحيح

### الخطوات:
```
1. دخول بأي حساب (Admin أو User)
2. اضغط على اسم المستخدم في الأعلى الأيمن
3. اضغط "Logout"
```

### النتيجة المتوقعة:
```
✅ حذف الجلسة
✅ حذف جميع البيانات المرتبطة بالمستخدم
✅ إعادة توجيه لصفحة الدخول
✅ رسالة نجاح: "Logged out successfully" (إن ظهرت)
```

### التحقق:
```
4. حاول الوصول إلى /admin/dashboard بدون دخول
اضغط Back عدة مرات
✅ يجب أن تعود إلى صفحة الدخول
```

## 📱 اختبار #10: Responsive Design

### الهدف:
التحقق من أن الصفحات تعمل على جميع الأحجام

### الخطوات:
```
1. دخول بأي حساب
2. افتح Developer Tools (F12)
3. جرب الأحجام المختلفة:
   - Mobile (320px)
   - Tablet (768px)
   - Desktop (1024px+)
4. تفقد كل صفحة
```

### التحقق:
```
✅ Navbar: يتحول لـ Hamburger على Mobile
✅ Dashboard: البطاقات تترتب بشكل صحيح
✅ Navbar Links: تظهر وتختفي بناءً على الحجم
✅ Tables: تكون Responsive (scroll إذا لزم)
✅ Forms: تأخذ عرض الشاشة
✅ لا توجد أخطاء أو تداخلات
```

## 🐛 اختبار #11: معالجة الأخطاء

### اختبار رسالة خطأ الدخول:
```

2. أدخل: test@test.com / wrongpassword
```
